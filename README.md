## DEVOPS-EXA
Projet du module devops

## Clone du repo
```sh
git clone https://github.com/MatheoDev/devops-exa.git
```

## Initialisation du projet avec Terraform
Stack technique :
- VueJS (frontend)
- NestJS (backend)
- PostgreSQL (bdd)
- Docker compose (local -> sur le port 3000 et 8080)
- Minikube (provider terraform - en local)
- Terraform (infra as code)
- Github Actions (on pull request - master branch)

Start du minikube
```sh
minikube start
```

Lancer les commandes suivantes pour initialiser le projet avec terraform dans un minikube

```sh
terraform init
terraform apply
```

## Cas particulier avec Minikube
Pour pouvoir avoir accès à l'appli sur notre local, nous devons exposer les ports des nodes.

Pour ce faire nous devons execter les commandes suivantes, dans 2 terminaux différent.

```sh
kubectl port-forward service/frontend -n devops 30001:80
kubectl port-forward service/backend -n devops 30002:3000
```

Cela permet d'avoir accès aux deux services dans notre navigateur.

## Navigateur

Maintenant dans le navigateur il suffit d'accéder à l'URL ``localhost:30001`` (frontend -> affichage des produits suite à un call API du backend) ou ``localhost:30002`` (backend -> Affiche un JSON ``{"test": "Bonjour tout le monde!"}``)
